# 工具与集成

> Last updated: 2026-05

## 概述

Chatbox Plus 通过工具系统扩展 AI 的能力，使 AI 不仅能生成文本，还能执行搜索、读取文件、查询知识库、调用第三方服务等操作。

> Chatbox Plus 是 GPLv3 fork，不内置上游 Chatbox AI 账号、订阅、license 或云端商业服务。涉及上游商业服务的旧设计在本 fork 中不作为可用能力承诺。

## 网页搜索与读取

启用网页浏览后，AI 可以在对话中调用两类工具：

- **网页搜索（Web Search）**：根据查询从互联网获取最新信息
- **读取网页（Read Webpage）**：解析指定 URL 的可读正文，用于深入理解某个网页

不同搜索提供方支持的工具能力有差异：

| 提供方 | 网页搜索 | 读取网页 | 说明 |
|--------|---------|---------|------|
| Bing | ✓ | ✗ | 免费使用，覆盖国际内容 |
| DuckDuckGo | ✓ | ✗ | 免费使用，隐私友好的通用搜索 |
| Tavily | ✓ | ✓ | 高质量 AI 搜索，需自备 API Key |
| BoCha | ✓ | ✗ | 国内搜索 API，需自备 API Key |
| Querit | ✓ | ✗ | 多源聚合搜索，需自备 API Key |

设置页（设置 → 网页搜索）会在所选提供方下方明确列出该提供方支持哪些工具，便于用户选择。

对 Bing 和 DuckDuckGo，设置页还提供“改善网络兼容性”开关，可将搜索请求通过 Chatbox 官方代理转发，并可直接检查当前搜索提供方是否可用。

使用方式：在会话中开启"网页浏览"模式，AI 在需要搜索或读取网页时会自动调用对应工具。如果当前提供方不支持读取网页（例如 Bing），AI 不会被注入 `parse_link` 工具，避免出现"调用了但失败"的情况。

### 工具失败时的引导

当工具因为缺少配置而失败（例如选择了 Tavily 但没有填写 API Key），消息中的工具气泡会自动展开，显示本地化的错误说明，并提供一键跳转链接（前往设置页输入 API Key 等），不再只是"工具调用失败"的红色提示。

## MCP 工具扩展

[MCP（Model Context Protocol）](https://modelcontextprotocol.io/) 是一种开放协议，允许 AI 通过标准化接口调用外部工具。

### 内置 MCP 服务

Chatbox Plus 不预置上游云端 MCP 服务。用户可以自行配置本地或远程 MCP 服务器：

| 服务来源 | 功能 |
|----------|------|
| 本地 MCP 服务器 | 由用户在本机运行，适合文件处理、自动化脚本等 |
| 远程 MCP 服务器 | 由用户配置 URL，适合团队或自有基础设施 |

### 自定义 MCP 服务器

用户可以添加自己的 MCP 服务器，扩展 AI 的能力边界。支持两种连接方式：

| 方式 | 说明 | 适用平台 |
|------|------|---------|
| 本地进程 (Stdio) | 在电脑上运行 MCP 服务器程序 | 仅桌面端 |
| HTTP | 通过网络连接远程 MCP 服务器 | 所有平台 |

## Agent Skills

Agent Skills 是基于 agentskills.io 规范的指令技能系统，适合把高频任务沉淀成可复用能力包（例如翻译、代码审查、写作、数据分析）。

产品侧提供两级控制：

- **全局级**：在设置页统一启用/禁用技能
- **会话级**：在输入框 Skills 菜单按会话覆盖启用列表

对话时，模型先获得技能目录元数据，再按需调用 `load_skill` 加载完整技能指令，避免把所有技能全文一次性注入上下文。

> Skills 当前为桌面端能力，Web/移动端不显示相关入口。

详见 [Agent Skills](./agent-skills.md)。

## 文件读取

用户在对话中上传的文件，AI 可以直接读取和分析：

- 小文件（500 行以内）：完整内容直接发送给 AI
- 大文件（超过 500 行）：AI 先看到文件摘要，需要时通过工具读取完整内容

支持上传代码、文档、数据文件等多种格式。

## 知识库查询

当会话关联了知识库时，AI 会自动获得知识库查询工具，可以：

- 对知识库进行语义搜索
- 查看文件列表和元信息
- 按块读取特定文档内容

详见 [知识库](./knowledge-base.md)。
